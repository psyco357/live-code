## Keputusan Teknis

* Menggunakan Next.js App Router karena merupakan framework yang paling saya kuasai.
* Menggunakan API Route agar arsitektur menyerupai REST API sederhana.
* Menggunakan file JSON sebagai penyimpanan data sesuai instruksi soal.
* Memisahkan business logic ke dalam `lead-service.ts` agar kode lebih mudah dipelihara.
* Menggunakan TypeScript untuk meningkatkan type safety.

## Jika Diberi Waktu Tambahan

Jika Diberi Waktu Tambahan

Beberapa hal yang ingin saya tambahkan untuk meningkatkan kualitas aplikasi adalah:

* Mengganti penyimpanan file JSON dengan database relasional seperti PostgreSQL atau MySQL.
* Menambahkan ORM atau data access layer agar pengelolaan data lebih terstruktur.
* Menambahkan validasi input yang lebih lengkap, baik di sisi client maupun server.
* Menambahkan Unit Testing dan Integration Testing untuk memastikan setiap fitur berjalan dengan baik.
* Mengimplementasikan tampilan yang mengikuti Salesforce Lightning Design System (SLDS) agar lebih menyerupai antarmuka Salesforce.
* Menambahkan autentikasi dan otorisasi sehingga hanya pengguna yang berhak yang dapat mengelola data Lead.
* Menyiapkan proses deployment menggunakan Docker agar aplikasi lebih mudah dijalankan di berbagai environment.
* Menambahkan fitur pencarian, filter, sorting, dan pagination untuk memudahkan pengelolaan data Lead dalam jumlah besar.
